Build a production-ready full-stack Voice AI agent called **“Jeevan Voice”** that improves accessibility for low-literacy users using voice-first interaction. The system must integrate Vapi (for voice AI) and Qdrant (for contextual memory and personalization), and be fully deployable on Replit.

## Core Requirements

* Voice-first AI agent (no complex UI needed, simple test interface is enough)
* Multilingual support (English, Hindi, Kannada at minimum)
* Context-aware responses using memory
* Real-world use cases: healthcare guidance, government schemes, financial assistance, and daily navigation

## Tech Stack

* Backend: Python (FastAPI)
* Vector DB: Qdrant (use cloud or local docker setup)
* Voice AI: Vapi (API integration)
* LLM: OpenAI-compatible API (use environment variables)
* Deployment: Replit (must run with one-click)

## Features to Implement

### 1. Voice Interaction (Vapi)

* Accept voice input via Vapi webhook
* Convert speech-to-text
* Send text to backend processing
* Return AI-generated response as speech

### 2. Context Memory (Qdrant)

* Store conversation embeddings per user
* Store metadata: language, preferences, past queries
* Retrieve relevant past context for each query
* Use semantic search for better responses

### 3. Decision Engine

* Detect user intent:

  * Healthcare (symptoms, urgency)
  * Finance (balance, schemes)
  * Government help
  * General queries
* Add rule-based layer for critical cases:

  * If emergency keywords (pain, chest, faint) → trigger urgent response mode

### 4. Personalization

* Remember:

  * User language
  * Past interactions
* Adapt tone and suggestions accordingly

### 5. API Endpoints

Create clean FastAPI routes:

* POST /vapi-webhook → main voice handler
* POST /query → text query (for testing)
* GET /health → health check

### 6. Qdrant Setup

* Create collection: "user_memory"
* Use embeddings for storing conversations
* Implement:

  * upsert_memory(user_id, text)
  * search_memory(user_id, query)

### 7. Response Generation

* Combine:

  * Current query
  * Retrieved memory
  * System prompt (empathetic, simple language)
* Generate conversational, low-literacy-friendly responses

### 8. Deployment Ready

* Use environment variables:

  * OPENAI_API_KEY

  * VAPI_API_KEY
